# Link3IT — Multi-Page Site (Next.js + Tailwind)

The identity-security consulting site for **LJSTECH LLC DBA Link3IT**, rebuilt as a
real multi-page application: App Router routing, a Tailwind design system, and a
markdown-driven blog. Static-exported for **Cloudflare Pages**.

## What's here

| Route | What it is |
|---|---|
| `/` | Home — hero, problem narrative, service + method + insights previews |
| `/services` | Services listing |
| `/services/[slug]` | One detail page per service (problem, approach, outcome, deliverables) |
| `/method` | The five-phase engagement method, in depth |
| `/insights` | Blog listing (reads markdown from `/content`) |
| `/insights/[slug]` | Individual article pages |
| `/about` | Founder + practice story (honest cert handling) |
| `/contact` | Consultation form (with "referred by") |

## Run it locally

```bash
npm install
npm run dev      # http://localhost:3000
```

## Build the static site

```bash
npm run build    # outputs to ./out
```

> The build downloads Google Fonts (Fraunces, Inter, JetBrains Mono) via `next/font`.
> It needs network access the first time — normal on your machine.

## Deploy to Cloudflare Pages

Push to GitHub, then in Cloudflare → Workers & Pages → Create → Pages → Connect to Git:
- **Build command:** `npx next build`
- **Output directory:** `out`
- **Framework preset:** Next.js (Static HTML Export)

Then add `link3it.com` under Custom domains.

**CRITICAL one-time step — activate the contact form:**
After the site is live, open `/contact` and submit the form once yourself.
FormSubmit emails **asymister@link3it.com** a confirmation link — click it once and
every future submission lands in your inbox forever. Do this immediately on launch so
the first real lead isn't missed.

The form is built so a lead is *never* silently lost: if the send fails for any reason,
it auto-opens the visitor's email app pre-filled to you, and your address is always
shown with a one-click copy button. Three guaranteed paths to your inbox.

## Publishing a new blog post

Drop a markdown file in `/content`. Frontmatter required:

```markdown
---
title: "Your title"
category: "CyberArk · PAM"
date: "2026-06-01"
readingTime: "5 min read"
excerpt: "One or two sentences shown in listings."
---

## A heading

Body text with **bold**, *italic*, `code`, [links](/x), lists, and > quotes.
```

It appears automatically on `/insights` (newest first) and gets its own page. No code changes.

## Editing services

All four services live in `lib/site.js`. Edit the array there and both the listing and
the detail pages update — they read from the same source.

## Security upgrade (recommended, post-launch)
The contact form posts to FormSubmit. To take the destination email out of client JS and
add rate limiting + Turnstile, add a Cloudflare Pages Function at `functions/api/lead.js`
and point the form's fetch at `/api/lead`. (Static export + Functions coexist.)
```js
export async function onRequestPost({ request, env }) {
  const data = await request.json();
  // validate, verify Turnstile, rate-limit, then forward via MailChannels/Resend
  return new Response(JSON.stringify({ ok: true }), { headers: { "content-type": "application/json" } });
}
```
